{# Shared CSRF fetch wrapper (P0-002). Included once per layout via `{% include "_csrf_fetch.html" %}`. Wraps window.fetch so that the X-CSRFToken header is attached ONLY to same-origin unsafe requests (POST/PUT/PATCH/DELETE). Cross-origin requests never receive the token, preventing leakage of the session-bound signed token to third parties. Behavior contract: * relative URLs and same-origin absolute URLs -> token attached; * cross-origin absolute URLs -> NO token (even for POST/PUT/PATCH/DELETE); * works for string URLs, URL objects and Request objects; * GET/HEAD never get a token; * a caller-provided X-CSRFToken is never overwritten; * body, credentials, return value and exception behavior are unchanged. #}
